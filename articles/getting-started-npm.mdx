כמה פעמים יצא לכם לעשות npm install “package-name”  וחשבתם וואלה גם אני הייתי רוצה. גם לי יש רעיון מגניב שהייתי רוצה לשתף אבל איכשהו חששתם? או אולי אמרתם אנחנו לא מספיק טובים כדי לשתף קוד?

אז בסדרת הכתבות הבאה נראה לכם עד כמה פשוט להרים חבילה  .

את הדרך שלנו נתחיל כאן
https://www.npmjs.com/signup
ההרשמה היא די פשוטה 

לאחר שנרשמתם צריך להקים את התשתית שתגדיר את החבילה שלכם. 
ניצור את הקובץ האלמותי package.json  אבל איך?
ניצור תקייה בשם pullrequestpackage  (אתם צריכים ליצור שם יחודי)
נכנס פנימה ונריץ npm init 
השאלה הראשונה היא מה שם החבילה – אחד הדברים החשובים ביותר שנכניס לPackage.json   שלנו. Npm init   בוחר את שם התקייה כברירת מחדל  לכן פשוט אלחץ Enter
https://docs.npmjs.com/cli/v8/configuring-npm/package-json#name
השאלה הבאה  היא בחירת הגרסה הראשונית. החשיבות מאחורי הגרסה היא עבור משתמשי החבילה. הגדרה טובה של הגרסה תבהיר עבורם איפה החבילה נמצאת מבחינה התפתחותית ועד כמה השינויים היו משמעותיים. לכן חשוב לעקוב אחרי חוקי הSEMVER 
https://semver.org/
מאחר והגרסה שלי היא מאוד ראשונית אני הולך לתת את הגרסה 0.0.1
השאלות הבאות הם 
Description  - מתאר את החבילה שלכם
Entry Point   - זוהיא נקודת ההתחלה של החבילה בpackage.json  תמצאו אותה תחת הmain
Test command   - הפקודה שתרוץ בהרצת הפקודה npm test
Git repository  - הURL  לבית הקיבול שלכם (גוגל טרנזלייט הוא כלי נהדר)
Keywords   - מילות מפתח שקשורים לחבילה 
Author   - השם של המיינטיינר. 
License    - הרשיון לחבילה 
לכולם נלחץ ENTER
נהדר השלב הראשון נגמר
עכשיו ניצור קובץ index.js 
לא נתמקד כאן על תוכן החבילה אבל נרצה לראות אותה עובדת
אז בקובץ נכניס את הקוד הבא 
export function getUser(userid){
    return "user not found"
}


עכשיו נרצה לדחוף את החבילה לNPM 
Npm publish

נכניס שם משתמש, סיסמה וכנראה שיבקשו גם OTP
